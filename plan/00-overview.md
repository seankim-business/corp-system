# Kyndof Corp System - 전체 개요

## 비전

**"회사의 구조와 프로세스를 코드가 아닌 자산으로 남기는 AI 기반 운영 시스템"**

기술은 빠르게 변하지만, 우리 회사의 Function/Value Stream/Agent/SOP 모델은 기술 스택과 독립적으로 유지되어야 한다.

---

## 유저 관점 요약

> **한 문장으로**: Slack에서 에이전트에게 말하면, 에이전트가 회사의 모든 도구와 문서를 연결해서 업무를 처리하고, 중요한 건 사람에게 확인받고, 결과를 알려준다.

### 유저가 할 수 있는 것

| 유저 요청 예시 | 시스템 반응 |
|---------------|------------|
| "다음 주 제품 런칭 마케팅 계획 세워줘" | 여러 에이전트가 협업하여 계획서 작성 |
| "A사 계약서 어디있어?" | Notion/Drive/GitHub/Slack에서 통합 검색 후 안내 |
| "신규 고객 온보딩 시작해줘" | SOP 로드 후 단계별 실행 (승인 요청 포함) |
| "Q1 OKR 진행상황 알려줘" | 실시간 현황 + AI 인사이트 제공 |
| "우리 온보딩 프로세스 정리해줘" | 기존 자료 분석 → SOP 초안 자동 생성 |

### 핵심 가치

```
┌─────────────────────────────────────────────────────────────────┐
│ 기존 방식                          →  Company OS               │
├─────────────────────────────────────────────────────────────────┤
│ 문서 찾으러 Notion, Drive 돌아다님  →  "어디있어?" 한마디면 끝   │
│ 반복 업무 매번 수동 처리            →  SOP 기반 자동 실행       │
│ 팀 간 협업 시 커뮤니케이션 오버헤드  →  에이전트가 알아서 협업   │
│ 프로세스가 구전으로 전달            →  SOP로 명문화, 버전 관리   │
│ 중요 결정 놓치거나 지연              →  승인 요청 알림, 추적     │
└─────────────────────────────────────────────────────────────────┘
```

> 📌 상세 유저 스토리는 [10-user-stories/user-stories.md](./10-user-stories/user-stories.md) 참조

## 도구 스택

| 도구 | 역할 | 특징 |
|------|------|------|
| **Notion** | 위키 + 프로젝트/태스크 관리 | 업무 실행, 칸반, 회의록, 가이드 |
| **Google Drive** | 시트/문서 관리 | 재무, 계약, 분석 리포트, 대외 문서 |
| **Figma** | 디자인/프로토타입 | 간헐적 사용 |
| **Slack** | 사내 커뮤니케이션 | 멀티 에이전트 협업 UI 중심 |
| **GitHub** | 모노레포 (SSOT) | 지식, SOP, 정책, 프로세스, 조직/역할 정의 |

---

## 핵심 원칙

### 1. Single Source of Truth (SSOT)
```
GitHub 모노레포 = 공식 지식/프로세스/SOP/조직 구조의 단일 진실 소스
```
- Notion/Drive는 **업무 실행 UI**로 사용
- 안정화된 문서만 GitHub으로 승격
- 에이전트는 GitHub을 기준으로 동작

### 2. 도메인 중심 설계
```
기술 스택 변경 → OK
도메인 모델 유지 → MUST
```
- Function, Value Stream, Agent, SOP가 핵심 자산
- 에이전트 엔진이 바뀌어도 도메인 모델은 그대로
- 설정 파일(YAML/JSON) 중심, 코드 최소화

### 3. 표준 기술 우선
```
많이 쓰이고 업그레이드가 잘 따라오는 것
```
- MCP (Model Context Protocol)
- OpenAI-style tools
- 표준 API, 대표적 에이전트 프레임워크
- 엔지니어 수가 적으므로 유지보수 용이성 중시

### 4. Human-in-the-Loop
```
중요 결정 = 사람 승인 필수
```
- 에이전트 자율 실행 + 명시적 승인 포인트
- PR 승인, 견적 확정, 인사/재무 변경 등
